# Science Guards (spec-layer scientific-rigor enforcement)

> Status: AUTHORITATIVE on the science guards (2026-06-22).
> Cross-ref: `design/evidence-validity.md` (the sibling referent-typed gate this
> composes with), `design/abstractions.md` (record vs belief; Spec/Evidence/Claim
> invariants), `design/literature-acquisition.md` (the novelty 2-kind that G1 reads).
> Source concern: a `formal` + `threshold` hypothesis can reach **SUPPORTED** while being
> (a) constructively true (a known theorem unit-tested as if discovered), (b) verified
> on a non-discriminating test set, or (c) checked by an apparatus that cannot report
> FAIL. None is caught by the evidence-validity gate (the data is genuinely `generated`,
> the referent is genuinely `formal`).

---

## 0. The concern these exist to prevent

The evidence-validity gate (`design/evidence-validity.md`) asks *can this DATA ground
this belief?* It correctly ALLOWS `formal` + `generated` evidence (T-1's molecule set
genuinely instantiates the encoding map). But it says nothing about whether the
**experimental design is strong**. Three weak-science patterns pass it untouched:

1. **Analyticity** — the hypothesis is true *by construction*. "A prime-power encoding
   round-trips" follows from unique factorization; a 100% round-trip is a known theorem.
   Stamping it "hypothesis SUPPORTED" packages a unit test as a discovery.
2. **Non-discriminating tests** — the test set is too easy. A pass over molecules that
   differ in atom counts says nothing about a canonicalizer; a plausibly-broken method
   passes it too, so the pass carries no information.
3. **Unfalsifiable apparatus** — nothing shows the test *can* report FAIL. If a method
   that should fail still passes, a SUPPORTED verdict is meaningless.

These are not data-validity failures; they are **claim-strength** failures. The science
guards add the missing layer.

---

## 1. The principle (and why it is a different layer)

Evidence-validity is a property of *evidence + referent* — checkable at the
claim-derivation primitive, declared via the frozen `referent`. **Claim strength is a
property of the RESEARCH INTENT** — whether this run is making a strong scientific claim
that must be falsifiable and discriminating. The kernel claim-derivation primitive runs
for every claim (including synthetic plumbing), so a HARD gate there would fire on the
primitive, not on the science.

So the guards split their enforcement across **two points** and a **declared strictness**:

- **Spec gate** (`core/spec_science.audit_spec_science`, called by
  `ResearchCompiler.stage_init_spec`): ALWAYS on, NEVER halts. Surfaces the structural
  findings as recording-type checkpoints (`checkpoints/science.json` + `science.md`), so
  a weak Spec is never SILENTLY accepted.
- **Verdict gate** (`core/validity.check_{analyticity,discriminating_power,
  falsifiability_adequacy}`, called by `ClaimUpdater._evaluate_hypothesis` right beside
  `check_evidence_adequacy`): HARD halts, ENFORCED only under `strict_science`.
- **`strict_science`** is the run-level declaration (mirroring how `referent` is the
  hypothesis-level declaration): **lenient at the primitive** (`ClaimUpdater` /
  `ResearchCompiler` / `verify_run` default `False` — a low-level caller is not blocked),
  **strict at the real research entrypoints** (`sci-adk run` / `derive-claim` default
  strict; `--no-strict-science` opts out; `verify --strict-science` opts in). A real
  research run thus refuses a weak SUPPORTED by default, while the build-harness unit
  tests that exercise the primitive with synthetic claims are unaffected.

The guards COMPOSE with the evidence-validity gates at the same chokepoint and never
weaken them — they only ADD refusal paths. They are PURE, no-LLM, no-domain: the kernel
reads frozen Spec fields + recorded Evidence. The deep judgments (is this case genuinely
hard? is this a known theorem?) are the author's, recorded for audit — the gate only
enforces that the DECLARATION it demands is present (the §5 honest-limit spirit, inherited
from evidence-validity).

## 2. The five guards

### G1 — Analyticity (verdict gate + spec surface)

**Trigger** (`check_analyticity`): `referent == formal` AND `decision_rule.kind ==
threshold` (deterministic, no RNG) AND `novelty_result` and `novelty_method` both False
(the result is already known in prior art) AND `epistemic_kind == "finding"` AND the
verdict is a binding SUPPORTS.

> Note: the trigger is **data_source-independent** (fail-closed). The original prose draft
> also required `data_source == generated`; that left a hole — a known-result `finding`
> whose Evidence had `data_source` unset/`None` would pass the verdict gate while the spec
> gate flagged it. A `formal` hypothesis's Evidence is generated by construction, so
> dropping the condition blocks no legitimate case and makes the verdict-gate G1 trigger
> identical to the spec-gate G1 trigger (`audit_spec_science`). (Closed after the
> independent review of 2026-06-22.)

**Action**: HALT. The author must either **reclassify** (`epistemic_kind` ->
`capability_check` for a capability assertion, or `unit_test` for a property true by
construction — the verdict framing becomes "capability verified / implementation correct",
not "hypothesis SUPPORTED") or **assert novelty** (`novelty_result` / `novelty_method`
with a recorded `found_nothing` prior-art search), recorded as a Spec amendment.

**Nuance (required)**: verifying an **open conjecture** by examples is legitimate science
("no counterexample up to N"). The novelty assertion is exactly the structural signal of
an open question, so a hypothesis that asserts novelty is NOT triggered. Only a
**known-result** (novelty both False) framed as a `finding` is refused. `formal` is not
illegitimate; `generated` is not illegitimate — a *known theorem unit-tested as a
discovery* is.

`epistemic_kind` is a frozen field, default `finding` (fail-closed): forgetting to
declare it can never silently weaken the gate.

### G2 — Test-power (verdict gate + spec surface)

**Trigger** (`check_discriminating_power`): `formal` + `threshold` + binding SUPPORTS with
no declared `discriminating_cases`.

**Action**: HALT (consistent with the chosen verdict-HARD-halt enforcement; a "passed
(low-power, non-discriminating)" downgrade was the considered alternative but would need a
new Claim status). The author declares `DiscriminatingCase` entries — hard cases that
SEPARATE a correct method from a plausibly-broken one, each with the reason. This is the
**G2<->G3 anchor**: the negative control must FAIL on exactly these cases.

### G3 — Falsifiability (verdict gate) — the most important

Mutation testing for science. **Trigger** (`check_falsifiability_adequacy`): `formal` +
`threshold` + binding SUPPORTS. **Requires** ≥1 `NEGATIVE_CONTROL` Evidence item bound to
the hypothesis satisfying ALL:

- `outcome == "not_supported"` — the mutant correctly FAILED the apparatus;
- **real execution provenance** — a non-empty `provenance.code_ref` or
  `provenance.environment` on the parent item (the mutant was actually RUN, not asserted);
- `discriminating_cases_covered` covers the declared `discriminating_cases` — the mutant
  fails ON THE HARD CASES (G2<->G3 coupling); a mutant that fails only an easy case proves
  nothing.

Missing/inadequate -> HALT ("supported (apparatus unfalsified)"). The control is a
`NEGATIVE_CONTROL` kind with `bears_on=[]` (a record ABOUT the apparatus, mirroring
`NOVELTY_DECISION`), so it NEVER enters the DecisionEngine — a mutant's refutation does not
contaminate the hypothesis verdict. Because it lives in the append-only Evidence log,
`verify --strict-science` re-derives it and `record_digest` covers it (tamper-evidence: a
deleted control makes a recorded strict SUPPORTED DIVERGE).

### G4 — Mode-coherence (spec lint)

**Trigger** (`audit_spec_science`): `decision_rule.kind == threshold` AND `mode ==
exploratory`. A frozen pre-registered pass/fail threshold belongs to a `confirmatory`
hypothesis; an exploratory hypothesis's rule should be a guide, not a binding gate.
Structural, surfaced (not a verdict halt). Resolution: set `mode = confirmatory` or use a
non-threshold rule, via amendment.

### G5 — Claim-cost (spec lint)

**Trigger** (`audit_spec_science`): a practical-property term (`index`, `efficient`,
`scalable`, `fast`, `compact`, `succinct`, `lightweight`, `practical`, `optimal`) in the
hypothesis statement or its target claims, with no declared cost metric (`cost_metrics`
empty AND the rule's `statistic` names no size/time measurement). Surfaced. Resolution:
declare the metric (e.g. integer bit-length for a Gödel/prime-power "index" — which
reveals the super-polynomial blowup), via amendment.

## 3. Schema additions

- `Hypothesis.epistemic_kind: finding | capability_check | unit_test` (frozen, default
  `finding`) — G1.
- `Hypothesis.discriminating_cases: list[DiscriminatingCase] | None` — G2/G3.
  `DiscriminatingCase = {case, why}`.
- `Hypothesis.cost_metrics: list[str] | None` — G5 manual override.
- `EvidenceKind.NEGATIVE_CONTROL` + `NegativeControl = {hypothesis_id, mutant, outcome,
  discriminating_cases_covered, statistic?, observed_value?}` (carried on
  `EvidenceItem.negative_control`, `bears_on=[]`) — G3.

All Spec fields are frozen (anti-HARKing): a result cannot be relabelled after seeing the
outcome.

---

## 4. Firing -> resolution (amendment)

Every guard's resolution path is the EXISTING amendment mechanism (Guard 2 precedent):
supply the missing artifact (`discriminating_cases`, `epistemic_kind`, `cost_metrics` via
`sci-adk amend-spec`; a `NEGATIVE_CONTROL` via `sci-adk append-evidence`) or a
justification, recorded as an `AmendmentReceipt`. No new resolution mechanism is added.

---

## 5. Honest limits

- The gates enforce that the DECLARATION each guard demands is present — NOT that a
  discriminating case is genuinely hard, that a negative control's mutant is genuinely
  violating, or that a result is genuinely a known theorem. Those are the author's
  judgments, recorded for audit. The gate makes the *honest* declaration load-bearing; it
  cannot detect a deliberately mislabelled one (the same residual trust every provenance
  field carries).
- The verdict-gate HALTS fire only on `formal` + `threshold` (the deterministic analytic
  class) — fail-OPEN elsewhere. Empirical claims are gated separately by
  `check_evidence_adequacy`; bayesian/interval/proof/qualitative rules (whose falsification
  is not a single mutated re-run) and non-binding verdicts are out of scope, so a
  legitimate empirical or judged result is never blocked.
- G5 is a precise keyword lint (no NLP): it does not fire on a claim that AVOIDS the
  practical-property word even when the property is implied. This is deliberate (no false
  positives); a narrowly-worded spec that makes no size claim correctly trips no G5.

## 6. Worked example: t1-godel(v4)

The real recorded spec (`~/research/t1-godel/runs/t1-godel/spec.json`, version 4) has two
`formal` + `threshold` + `exploratory` hypotheses, novelty both False, and (predating these
guards) no `epistemic_kind` / `discriminating_cases` / `cost_metrics`:

- **hyp-001 (H1, injectivity, `collision_count == 0`)** and **hyp-002 (H2, recoverability,
  `roundtrip == 100%`)**.

Guards on:

- **Spec gate** surfaces, on BOTH hypotheses: **G1** (formal+threshold+finding+no-novelty —
  H2 is decided by unique factorization, a known theorem -> should be `unit_test`; H1's open
  part is the canonicalizer's correctness, not the prime-power packing -> restate),
  **G2** (no discriminating cases — the easy near-misses do not separate a correct
  canonicalizer; cospectral / EC-degenerate non-isomorphic pairs are needed), the **G3**
  reminder (a strict SUPPORTED needs a negative control: remove a tie-breaking invariant
  and confirm collisions; corrupt the decoder and confirm round-trip < 100%), and **G4**
  (exploratory + frozen threshold).
- **Verdict gate** (`strict_science`) HALTS each hypothesis (G1 first, then G2, then G3).
- **G5** does NOT fire on the literal v4 statements (they are narrowly worded — only
  `collision_count` and round-trip), which is correct; it fires immediately on the natural
  "compact molecular index" restatement the proposal implies, surfacing the prime-power
  bit-length blowup.

Tests: `tests/test_science_guards.py`.
